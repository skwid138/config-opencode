# Out-of-Scope Memory

Use this when an enhancement is rejected and future triage should not re-suggest it.

Per-project memory lives in the target repo under `.out-of-scope/`. Writes are Aragorn-only and user-visible.

Start every posted GitHub comment with:

```md
> *This was generated by AI during triage.*
```

## File template

```md
# {Rejected enhancement title}

## Summary

What was requested.

## Decision

Rejected.

## Rationale

Why this is out of scope. Capture the maintainer's reasoning, not agent speculation.

## Similar future requests

How to recognize duplicates or near-duplicates.

## Source

- GitHub issue: {url}
- Decision date: {yyyy-mm-dd}
```

## Comment template

```md
> *This was generated by AI during triage.*

Thanks for the suggestion. We're closing this as out of scope for now.

Reason: {brief rationale}

I've recorded this decision in `.out-of-scope/{slug}.md` so future similar requests are handled consistently.
```

Do not create `.out-of-scope/` entries for temporary deferrals. Use it only for durable rejected enhancements.
