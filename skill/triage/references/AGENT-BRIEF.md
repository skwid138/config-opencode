# AFK Agent Brief

Use this template when a GitHub issue is ready for an AFK agent.

Start every posted comment with:

```md
> *This was generated by AI during triage.*
```

## Template

```md
> *This was generated by AI during triage.*

## Agent Brief

**Goal:**
What the agent should accomplish in one or two sentences.

**Why this matters:**
User impact, maintainer goal, or regression risk.

**Context gathered:**
- Relevant files, modules, commands, docs, or prior comments.
- Reproduction result for bugs, if available.
- Constraints from ADRs or project conventions.

**Expected behavior:**
- Observable outcome 1.
- Observable outcome 2.

**Suggested approach:**
- First investigation or implementation step.
- Important edge cases.
- Known non-goals.

**Verification:**
- Focused tests or commands to run.
- Manual checks if needed.

**Risks / gotchas:**
- Anything likely to surprise the implementing agent.
```

Keep briefs durable: they should still make sense to a fresh agent with no access to the triage conversation.
